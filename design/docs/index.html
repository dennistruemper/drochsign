Getting started

Drochsign is a semantic CSS design system: each theme is a background and a foreground, like paper and ink. State and hierarchy use size, weight, and motion — not extra colors.

Quick start

Link the stylesheet and set a theme on <html>:

<link rel="stylesheet" href="drochsign.css" />
<html lang="en" data-theme="paper">

Switch themes in JavaScript (see Theme editor for custom themes):

document.documentElement.dataset.theme = 'ink';

Fonts

The default stack is set on :root as --font-body and --font-mono. Built-in themes can override these — try sepia (serif) or gamegirl (monospace).

Custom values — use the Theme editor or set tokens in JS:

const root = document.documentElement;
root.style.setProperty('--color-bg', '#fef08a');
root.style.setProperty('--color-fg', '#14532d');
root.style.setProperty('--font-body', 'Georgia, serif');

Elm

Add drochsign.css to your index.html. Use native elements in Html — button, article, input, etc. Set the theme on the root: 

Html.Attributes.attribute "data-theme" "paper"

Foldkit

Same approach: one CSS file, semantic markup in templates, and data-theme on <html>. No component classes required.
